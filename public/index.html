🪟 Windows 安裝指南（Standalone 本機版）

本節說明如何在 Windows 電腦上從零開始安裝並執行 YouTube Downloader。建議使用 Windows 11 / Windows 10（版本 1903+）。

✅ 方式 A：WSL2（強烈推薦）

透過 WSL2 在 Windows 內執行 Linux 環境，最接近原生體驗，支援所有功能（包括 Google Drive 上傳）。

  1. 1 安裝 WSL2：以管理員身份執行 PowerShell，輸入：wsl --install → 重啟電腦
  2. 2 設定 Ubuntu：重啟後 Ubuntu 自動開啟，設定用戶名及密碼
  3. 3 安裝 Homebrew（WSL 內）：
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  4. 4 安裝所有依賴：
    brew install node yt-dlp ffmpeg cloudflare/cloudflare/cloudflared steipete/tap/gogcli
  5. 5 Clone 及啟動：
    git clone <repo> youtube-downloader && cd youtube-downloader
    cp .env.example .env → 編輯 .env 填入帳號密碼
    npm install && bash start.sh
  6. 6 瀏覽器開啟：Windows 瀏覽器直接輸入 http://localhost:3847 即可存取（WSL2 自動共享端口）
💡 WSL2 與 Windows 共享 localhost，無需額外配置即可在 Windows 瀏覽器存取。
⚙️ 方式 B：原生 Windows（無 WSL2）

直接在 Windows 安裝所需工具。注意：Google Drive 自動上傳功能在原生 Windows 下不支援（gog CLI 為 macOS/Linux 限定）。

① 安裝必要工具（以管理員 PowerShell 執行）
# 安裝 winget 管理的工具
winget install OpenJS.NodeJS.LTS
winget install yt-dlp.yt-dlp
winget install Gyan.FFmpeg
winget install Cloudflare.cloudflared
# 安裝完成後重新開啟 PowerShell（刷新 PATH）
② 配置 .env 工具路徑

Windows 的工具路徑與 Linux 不同，需要在 .env 中指定：

# .env 新增（路徑視乎安裝位置而定）
YT_DLP_PATH=C:\Users\%USERNAME%\AppData\Local\Microsoft\WinGet\Links\yt-dlp.exe
FFMPEG_PATH=C:\ProgramData\chocolatey\bin\ffmpeg.exe
③ 啟動伺服器
# 方法 1：雙擊 start.bat（最簡單）
start.bat
# 方法 2：PowerShell / cmd 手動執行
cd youtube-downloader
npm install
node server.js
停止伺服器：雙擊 stop.bat
⚠️ 原生 Windows 限制：
  • Google Drive 自動上傳不支援（gog CLI 為 macOS/Linux 限定）
  • Telegram 通知功能需要額外配置 openclaw
  • 建議搭配 Cloudflare Quick Tunnel 取得公開存取 URL
💡 Windows ffmpeg PATH 設定： winget 安裝的 ffmpeg 可能不在 PATH 內。若下載時出現「ffmpeg not found」，需手動將 ffmpeg 安裝目錄加入系統 PATH，或在 .env 中設定 FFMPEG_PATH。
🔐 登入 & 帳號管理
  1. 1 登入：輸入用戶名（電郵格式）及密碼，點擊 LOGIN。Session 有效期 24 小時，到期後需重新登入。
  2. 2 忘記密碼：點擊登入頁的 Forgot Password? → 輸入註冊電郵 → 系統發送 6 位數驗證碼 → 輸入驗證碼 → 顯示當前密碼（驗證碼 10 分鐘內有效，最多 5 次嘗試）
  3. 3 修改密碼： → 「帳號安全」→ 輸入目前密碼 + 新密碼 + 確認新密碼 → 儲存。儲存後所有登入 Session 自動清除，需重新登入。
  4. 4 登出： → 點擊「🚪 登出」按鈕
⚠️ 密碼儲存在伺服器環境變數，請勿將帳號資訊分享給他人。
📁 分類管理
  1. 1 新增分類：在輸入框輸入分類名稱（如「AI」、「音樂」）→ 點擊 + 新增。分類會建立對應的 Google Drive 子資料夾。
  2. 2 修改名稱：點擊分類卡片上的 ✏️ 圖示 → 直接在卡片上修改名稱 → 點擊 ✅ 確認。
  3. 3 刪除分類：點擊 🗑️ 圖示 → 確認刪除彈窗。注意：「未分類」為系統預設分類，無法刪除。
  4. 4 拖曳排序：按住左側 ⠿ 拖曳手柄，上下拖動即可重新排序分類。排序會即時同步至伺服器。
💡 分類會顯示在下載頁的下拉選單中，同時對應 Google Drive 的子資料夾結構。建議先建好分類再開始下載。
⬇️ 下載影片（完整流程）
  1. 1 選擇分類：在頁面左上角的下拉選單選擇目標分類（如「AI」）。此選擇決定影片儲存的 Google Drive 資料夾。
  2. 2 貼上 YouTube 網址：在輸入框貼上完整 YouTube 影片網址（支援 youtu.be 短網址及 youtube.com 長網址）。亦可直接按 Enter 觸發分析。
  3. 3 點擊「分析」：系統呼叫 yt-dlp 分析影片元數據（約 3-10 秒）。分析成功後顯示：影片縮圖、標題、頻道名稱、時長，以及各畫質選項與預計檔案大小。
  4. 4 選擇封裝格式：在「格式」區選擇 MP4 MKV 或 MOV。三者均為無損封裝（不重新編碼），檔案大小相同，建議使用 MP4（兼容性最佳）。
  5. 5 選擇畫質：在「畫質」區點選所需解析度（如 1080p）。右側顯示預計檔案大小（影片 + 音訊串流合計）。可用畫質取決於原片，最高可至 2160p（4K）。
  6. 6 日期前綴（選填）：若影片有上傳日期，頁面會顯示開關。啟用後檔名格式為 YYYYMMDD-標題.mp4（如 20260308-影片標題.mp4），方便按日期排序。預設開啟。
  7. 7 點擊「開始下載」：系統在背景啟動下載，頁面自動跳轉至「管理」Tab 追蹤進度。
💡 下載過程：yt-dlp 分別下載影片串流 + 音訊串流 → ffmpeg 合併 → 自動上傳 Google Drive → 發送 Telegram 通知。全程無需干預。
⚠️ 部分影片（年齡限制 / 地區限制）需要 YouTube Cookies 才能下載。請到 上傳 cookies.txt。
📊 下載管理
  1. 1 下載狀態：管理頁分為「進行中」與「已完成」兩個區域，自動每 2 秒刷新。
    狀態流程：等待中 → 下載中 → 合併中 → 上傳 Drive → 完成 / 失敗
  2. 2 下載中：顯示即時進度條、下載速度（如 5.2 MB/s）、預計剩餘時間（ETA）及已下載大小。
  3. 3 完成後：顯示 Google Drive 連結，點擊 🔗 Drive 即可在瀏覽器開啟並播放影片。
  4. 4 重試失敗：下載失敗的記錄會顯示錯誤原因及 ↻ 重試 按鈕，點擊自動使用原設定重新下載。
  5. 5 刪除記錄：點擊 🗑️ 可刪除下載記錄（不刪除本地檔案及 Google Drive 上的檔案）。
💡 Telegram 也會自動收到通知，無需一直留在管理頁等候。
📋 活動日誌
  1. 1 概覽統計：頁面頂部顯示日誌總數、今日數量、成功 / 錯誤計數。
  2. 2 篩選日誌：可按「類別」篩選：下載 上傳 分析 認證 分類 系統；可按「等級」篩選：INFO / SUCCESS / WARN / ERROR。
  3. 3 查看詳情：點擊任意日誌條目展開詳細資訊（如下載 URL、錯誤訊息、Drive 連結等）。
  4. 4 載入更多：日誌每次顯示 50 條，點擊底部「載入更多」可載入更舊的記錄（最多保存 500 條）。
  5. 5 清除日誌：點擊右上角「清除」按鈕可清空所有日誌記錄（不可恢復）。
⚙️ 系統設定
  1. 1 字體大小：選擇「中」（預設 16px）、「大」（~19px）或「特大」（~22px）。設定即時生效並永久保存，適合不同螢幕尺寸。
  2. 2 YouTube Cookies：如需下載年齡限制或會員內容，需上傳從瀏覽器匯出的 cookies.txt。
    匯出方法：安裝 Chrome 擴充 Get cookies.txt LOCALLY，登入 YouTube 後點擊匯出。
  3. 3 修改帳號：輸入目前密碼 → 填寫新用戶名及/或新密碼（兩次確認）→ 點擊「💾 儲存帳號設定」。所有 Session 將立即失效。
  4. 4 登出：點擊「🚪 登出」按鈕，Session 即時失效並跳轉至登入頁。
  5. 5 Telegram 通知：填寫 Group ID 及（可選）Topic ID → 點「💾 儲存設定」→ 開啟啟用開關。詳見 📲 Telegram 通知設定指南。
☁️ Cloudflare Tunnel 設定

Cloudflare Tunnel 讓你可透過公開 URL 從外部（或手機）訪問在本機執行的 YouTube Downloader，無需開放防火牆或做端口轉發（Port Forwarding）。

① 先安裝 cloudflared
  1. 1Windows：winget install Cloudflare.cloudflared （以管理員 PowerShell 執行，安裝後重新開啟終端）
    或前往 官網 下載 cloudflared-windows-amd64.exe，改名 cloudflared.exe 並加入 PATH
  2. 2macOS：brew install cloudflare/cloudflare/cloudflared
  3. 3Linux：sudo apt install cloudflared
  4. 4安裝完成後回到 → Cloudflare Tunnel → 點「🔄 刷新」確認狀態顯示「已安裝」
② 方式 A：Quick Tunnel（推薦新手，免帳號）
  1. 1在設定頁選擇「⚡ Quick Tunnel」模式
  2. 2確認本地端口為 3847（預設值）→ 點擊 🚀 啟動 Tunnel
  3. 3狀態變為「運行中」後，查看「Tunnel 日誌」欄位，找到 *.trycloudflare.com 形式的公開 URL
  4. 4用任何裝置或瀏覽器輸入該 URL，即可遠端使用 YouTube Downloader
⚠️ Quick Tunnel 的 URL 每次重啟都會改變，不適合固定分享。如需固定 URL，請用方式 B。
② 方式 B：Token Tunnel（固定自訂域名）
  1. 1登入 Cloudflare Zero Trust → Networks → Tunnels → 點擊「Create a tunnel」→ 選 Cloudflared
  2. 2輸入 Tunnel 名稱（如 yt-downloader）→ 在 Install connector 頁面複製以 eyJ 開頭的 Token
  3. 3在 Public Hostname 設定中：Service 填 http://localhost:3847，Domain 填你的自訂域名（需要先在 Cloudflare 管理 DNS）
  4. 4回到 → Cloudflare Tunnel → 選「🔑 Token Tunnel」→ 貼上 Token → 啟動
💡 Token Tunnel 支援固定自訂域名（如 yt.yourdomain.com），URL 不會因重啟而改變。
📂 Google Drive 設定

Google Drive 整合透過 gog CLI 實現，下載完成後自動上傳至你的 Drive 並附上連結。以下是首次設定的完整流程。

🪟 Windows 用戶須知
gog CLI 僅支援 macOS / Linux，Windows 原生環境無法使用 Google Drive 自動上傳功能。
推薦方案：使用 WSL2 執行整個應用（參閱 Windows 安裝指南），可完整支援所有功能。
替代方案：下載仍可正常運作，但完成後不會自動上傳 Drive，影片存儲在本機的下載目錄。
① 安裝 gog CLI（macOS / Linux / WSL2）
  1. 1打開終端（macOS Terminal、Linux bash、或 WSL2 Ubuntu），執行：brew install steipete/tap/gogcli
  2. 2驗證安裝：gog --version，應顯示版本號
  3. 3回到 → Google Drive → 點「🔄 刷新」確認狀態不再顯示「未安裝」
② 授權 Google Drive
  1. 1點擊 🔗 開始授權，伺服器端的 gog auth login 會自動執行
  2. 2你的預設瀏覽器會自動打開 Google OAuth 頁面（本機模式下直接打開；若遠端存取，需在伺服器所在機器上操作）
  3. 3選擇你的 Google 帳號 → 點擊「允許」授予 Google Drive 存取權限
  4. 4瀏覽器顯示授權成功訊息後，回到此頁點擊 ✅ 已完成，刷新狀態
  5. 5狀態顯示「已連接 Google Drive ✓」即設定完成
💡 授權後的 Token 存儲在本機 ~/.config/gog/，重新安裝系統或清除該目錄後需要重新授權。
⚠️ 若授權後狀態仍顯示未授權，可能是 gog 版本較舊，嘗試執行 brew upgrade gogcli 後再重試。
③ 測試 Google Drive 整合
  1. 1前往 ，下載任意一條 YouTube 影片
  2. 2下載完成後，在管理頁會顯示「📁 Google Drive ↗」連結 — 點擊確認影片已上傳至 Drive
  3. 3你的 Google Drive 中會出現「Youtube Downloader」資料夾，按分類建立子資料夾
📲 Telegram 通知設定

設定後，每次下載完成都會自動在指定 Telegram 群組發送通知，包含影片標題、畫質及 Google Drive 連結。

🔧 前提條件
  • 伺服器上已安裝 OpenClaw CLI（openclaw 指令可用）
  • OpenClaw 已完成 Telegram Bot 設定及授權
  • Bot 已加入目標 Telegram 群組並具備發言權限
① 取得 Group ID
  1. 1打開 Telegram，前往你的目標群組
  2. 2將 @userinfobot 加入群組
  3. 3在群組發送任意訊息，Bot 會自動回覆群組資訊，其中 id 欄位即 Group ID（通常以 -100 開頭）
  4. 4複製該數字 ID，前往 → Telegram 通知 → 貼到「GROUP ID」欄位
② 取得 Topic ID（Forum 群組才需要）
  1. 1如果你的群組是 Forum 群組（即有話題/Thread 功能），需要額外填寫 Topic ID
  2. 2在 Telegram 中，右鍵點擊目標話題 → 複製連結
  3. 3連結格式如：https://t.me/c/1234567890/191，最後的數字即 Topic ID
  4. 4將 Topic ID 填入「TOPIC ID」欄位（不需要的話留空即可）
③ 儲存設定並測試
  1. 1填寫 Group ID（及可選的 Topic ID）後，點擊 💾 儲存設定
  2. 2點擊 📨 發送測試訊息，確認 Telegram 群組收到測試通知
  3. 3測試成功後，開啟「下載完成後自動發送 Telegram 通知」開關
  4. 4之後每次下載完成，系統會自動發送通知，無需手動操作 ✅
💡 通知內容包含：影片標題、畫質、完成時間，以及 Google Drive 分享連結（如已啟用 Google Drive）。
⚠️ 若測試失敗並顯示「openclaw: command not found」，表示伺服器未安裝 OpenClaw CLI，請聯絡管理員安裝後再試。
❓ 常見問題
  1. Q 分析失敗，提示「Sign in to confirm you're not a bot」
    → 需要上傳 YouTube Cookies。前往 上傳 cookies.txt。
  2. Q 下載後看不到 Google Drive 連結
    → Drive 上傳可能失敗。查看 的「上傳」類別，確認錯誤原因（常見：Drive 容量已滿）。
  3. Q 下載速度慢 / 被限速
    → 屬於 YouTube 伺服器端限速，為正常現象。高畫質影片（4K / 1080p）需要較長下載時間。
  4. Q 選擇的畫質（如 4K）不在列表中
    → 原始影片可能沒有該解析度。yt-dlp 只列出影片實際提供的畫質選項。
  5. Q 網站出現 Cloudflare Error 1033
    → Cloudflare Tunnel 服務中斷，請通知管理員重啟 Tunnel。
  6. Q MP4 / MKV / MOV 有什麼分別？
    → 三者均為封裝格式（container），影音內容完全一樣、檔案大小相同，只是「外殼」不同。MP4 兼容性最佳（Windows / macOS / iOS / Android 均支援），建議優先選用。
